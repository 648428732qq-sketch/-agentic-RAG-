# Query Translator 训练与冻结集回归（2026-06-30）

## 结论

本轮已完成离线训练、未见术语开发集评估和 V2 冻结集回归。按当前项目范围，17 条地域方言题已归档排除，有效冻结集为 83 条；训练过程未读取开发集或冻结集。

最终准入结论：**不替换当前 BGE 基座模型**。

- V1 普通正负对余弦回归发生灾难性遗忘，已淘汰。
- V2 使用“唯一术语批次对比学习 + 冻结基座蒸馏”，开发集明显提升，但冻结集 Top-10/Top-20 回退。
- 固定等权 RRF 融合改善了 MRR，但仍未满足冻结集全指标不回退要求。
- 两个检查点均未接入线上 RAG，rerank 仍保持关闭。
- 地域方言不再生成、不参与训练或准入分母；历史题仅可通过显式 `--include-dialect` 重放。

## 失败诊断

V1 把随机负例余弦相似度直接压向 0，破坏了基座原有中文语义空间：开发集 Recall@10 从 87.10% 跌至 6.91%。

V2 解决了灾难性遗忘，并证明现有监督对“训练分布内的未见术语”有效；但训练数据主要是规范词的直接证据匹配，缺少真实隐喻、残句、ASR 错字到规范术语的成对监督。因此它在冻结集上只改善靠前排序，无法稳定提高多症状完整召回。

分风格看，ASR 错字 Recall@5 从 29.23% 提升到 33.85%，隐喻/残句却从 42.42% 降到 37.88%。下一轮应补隐喻/残句训练对，并保留独立新终验集；不再投入地域方言数据。

## 下一准入条件

1. 增加不来自当前冻结集种子的普通口语/隐喻/残句/错字到规范术语成对数据；地域方言不纳入当前范围。
2. 重新划分 train/dev，并另建从未用于调参的新冻结集。
3. 新模型必须同时满足：开发集提升、冻结集 MRR 与 Recall@5/@10/@20 全部不回退、否定与需追问安全指标不回退。
4. 达标后才接入 `SyndromeRetriever`；接入后再跑端到端 Qdrant 检索、门控和拒答回归。

本轮结果不支持继续用同一冻结集反复调参，否则会把冻结集变成开发集并产生评测过拟合。
