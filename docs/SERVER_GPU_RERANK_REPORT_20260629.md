# 服务器 GPU Rerank 实测报告

## 环境隔离

- Python：3.10.12，项目内 `.venv-linux`
- PyTorch：2.11.0+cu128
- GPU：单张 NVIDIA A100-SXM4-80GB，通过 `CUDA_VISIBLE_DEVICES` 隔离
- 模型：`BAAI/bge-small-zh-v1.5`、`BAAI/bge-reranker-v2-m3`
- 系统 CUDA、全局 Python 和其他用户环境均未修改
- 远程完整回归：132 / 132
- CUDA、UTF-8、模型离线缓存检查：通过

## 400 条 Hard Negative A/B

数据覆盖 109 个可测方剂，包含完整方证、缺少必要症状、禁忌冲突和共有症状四类场景。

| 指标 | 本地硬排序 | GPU rerank |
|---|---:|---:|
| 通过数 | 400 / 400 | 400 / 400 |
| 平均延迟 | 436.11 ms | 534.64 ms |
| P95 延迟 | 548.33 ms | 653.58 ms |
| rerank 实际调用 | 0 | 400 |
| rerank 错误 | 0 | 0 |

GPU rerank 平均增加 98.53 ms，P95 增加 105.25 ms。Top-1 改变 150 条，Top-K 顺序改变 375 条；目标名次改善 94 条、变差 17 条、不变 289 条。

- 完整方证 109 条：Top-1 全部不变，准确率无新增收益。
- 缺少必要症状 109 条：目标名次改善 54 条、变差 2 条。
- 共有症状 109 条：目标名次改善 40 条、变差 15 条。
- 禁忌冲突 73 条：安全门控无退化。

## 端到端探针

3 条 DeepSeek + 内存 Qdrant + GPU rerank 探针未达到验收：术语召回 66.67%，检索 33.33%。失败暴露的是上游契约和跨来源检索问题，不是 CUDA 模型不可用：

1. “恶寒、身痛、无汗但未说明喘”的样本，金标准要求直接回答；当前安全门控选择追问，但麻黄汤和九味羌活汤均已进入 Top-5。该金标准与“缺关键鉴别项必须追问”的安全规则冲突，应重新审核，不能为了通过测试放宽门控。
2. 胃胀痛且缺少“食饮不下”的样本正确触发追问，但针灸条文未进入 Top-5，说明临床症状查询仍偏向方剂来源，需要增加跨 `source_type` 的候选多样性。
3. ASR 错字样本依赖 DeepSeek 复核；服务器连接不稳定，且 0.70 置信度的本地候选映射被 0.72 阈值拒绝。应先建立通用的 ASR/同音归一化与有证据的低置信映射策略，再用未见测试集验证。

## Qdrant 结论

项目位于 NFS 共享存储。最小 SQLite 和 Qdrant Local 写入均复现损坏，分别出现 `database disk image is malformed` 与 `disk I/O error`。因此：

- 不使用复制来的 Windows `qdrant_db` 作为 Linux 活动索引。
- 本轮评测使用进程内 Qdrant，共 1419 点，退出即释放。
- 正式 WebUI 必须连接独立 Qdrant 服务，服务端 storage 放在本地块存储；在获得本地存储目录权限前不部署持久索引。

## 决策

保持 `ENABLE_SYNDROME_RERANK=false`。GPU rerank 链路已验证可用，能改善部分模糊候选的目标名次，但未提升现有 400 条测试的通过率，并产生 17 条名次退化。下一轮应先完成独立、未参与调参的分级排名金标准和跨来源候选多样性，再决定是否仅对“缺项追问候选”启用二级重排。
