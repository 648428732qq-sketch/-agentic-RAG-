# 金标准完整性复核

## 结论

当前 `query_translator_fresh_final_100` 不是可用于对外泛化结论的最终盲测集，应降级为回归集。问题是评测过程的数据泄漏和适应性过拟合，不代表 DeepSeek 或 reranker 在参数层面记住了答案。

## 证据

1. 同一 100 条先后生成原始报告以及 `after_fix1` 到 `after_fix5` 报告，分数从 51/100 上升到 93/100。每轮都查看同一批失败后继续修改规则，违反最终留出集只能运行一次的约束。
2. 文档明确列出针对该批样本补入的同音错字和口语模式，例如“发扫、吐玩、经慢、瓶躺、谈像清西”。这些改动可以作为开发集改进，但对应数据不能继续称为未见盲测。
3. 3 条 `rerank_probe` 全部是原 100 条金标签的逐字段相同副本，不是新生成样本。
4. `questions_rerank_probe.jsonl` 创建于 20:31:52，预测在 20:33:33 至 20:34:34 生成，probe 私有标签文件在 20:34:57 写入。即使标签来自原始 100 条，子集选择也发生在整批结果已经被反复观察之后，存在明显选择偏差。
5. 旧的 3/3 预测中，“无汗”样本同时输出“汗出”和“无汗”，旧评分器仍判通过。原因是评分器只检查金标签显式提供的 `forbidden_terms`，没有检查期望表征的互斥冲突。
6. 该批标签的 `review_status` 为 `qwen_generated_structurally_validated`，没有人工临床复核，因此准确说法应是“合成银标准”，不是临床金标准。

加入期望表征互斥检查后，对原预测和原标签重新评分，历史结果从 3/3 降为 2/3；失败项正是同时输出“无汗”和“汗出”的样本。复评分报告为 `datasets/structured/query_translator_fresh_final_100_report_rerank_probe_integrity_rescore_20260629.json`。

## 本次失败的组成

- **评测过拟合**：规则和测试已针对固定 100 条反复调整，这是主要完整性问题。
- **标签契约冲突**：仅有恶寒、身痛、无汗而缺少喘等鉴别项的样本，标签要求直接回答；当前安全门控选择追问。不能为了旧分数放宽安全规则。
- **外部模型不稳定**：服务器访问 DeepSeek 存在 TLS 握手和 60 秒请求超时；同一问题的结构化翻译可能不同。
- **索引运行差异**：旧报告使用持久化本地 Qdrant，本次 NFS 环境只能使用从同一 JSONL 临时构建的内存 Qdrant，候选并列顺序可能变化。
- **评分器过宽**：只要求答案位于 Top-5，没有分级相关性、MRR 或 nDCG，也不惩罚部分错误 Top-1。

因此，这次失败不能简单归因于 reranker，也不能用旧 3/3 证明当前实现回归。

## 后续验收协议

1. 将现有 100 条和 3 条 probe 固定为 `regression_only`，可以持续跑，但禁止用于最终泛化声明。
2. 先冻结 Query Translator、候选融合、硬门控、rerank 配置和评分器版本并记录 Git commit。
3. 从未进入现有种子和单元测试的新方证、新问法生成独立测试集；公开问题与私有标签分离保存。
4. 生成后先做自动一致性审计：标签证据必须来自本地 payload，互斥症状不得同时为正，缺关键鉴别项必须标记追问。
5. 不根据最终集逐题补词或改规则。一次性报告 Recall@5、MRR@10、nDCG@10、拒答/追问正确率、安全冲突率和 P95 延迟。
6. 若运行后继续调参，该数据集立即降级为回归集，并重新生成下一批真正未见数据。
