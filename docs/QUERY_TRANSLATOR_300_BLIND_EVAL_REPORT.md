# Query Translator 300条混合盲测报告

## 测试范围

- 测试日期：2026-06-22
- 基础金标准：16条本地结构化检索用例
- 扩展盲测：300条Qwen生成的表达变体
- 生成模型：`qwen3.6-flash`
- 被测翻译模型：DeepSeek `deepseek-chat`
- 表达类型：方言、隐喻/不完整句、ASR错字、否定/反问、多症状长句、Hard Negative
- 每种类型：50条
- 语义种子族：16个；300条是表达泛化测试，不是300个独立医学病例

公开问题文件只包含`id/query`。在线预测进程不读取私有金标准；规范术语、极性、禁词、门控和检索目标仅在离线评分进程使用。

## 300条盲测结果

| 指标 | DeepSeek受约束翻译 | 纯本地降级链 |
|---|---:|---:|
| 全项通过 | 220/300（73.3%） | 6/300（2.0%） |
| 规范术语召回 | 81.0% | 8.33% |
| 候选词覆盖 | 100% | 100% |
| 否定识别 | 96.33% | 79.33% |
| 禁止词不误判为阳性 | 98.67% | 85.67% |
| 检索目标命中 | 100% | 100% |
| 回答/追问门控 | 93.33% | 29.0% |
| 必须追问准确率 | 97.33% | 未单列 |
| 平均延迟 | 4.89秒 | 0.92秒 |

DeepSeek受约束翻译将规范术语召回提高72.67个百分点，同时保留本地术语白名单、证据片段校验和数据库外拒答约束。

## 主要失败

- 规范术语漏召回：57条。
- 门控决策错误：20条。
- 否定症状漏识别：11条。
- 禁止症状被错误识别为阳性：4条。
- 必须追问但未追问：8条。

高频漏召回包括：`吐后`8次、`恶风`7次、`十二经`6次、`腹䐜胀/腹胀满`6次、`明目`5次、`食饮不下`4次。

门控混淆：

- 应直接回答却返回追问：11条。
- 应追问却直接回答：8条。
- 应追问却返回无匹配：1条。

## 结论

当前系统已证明受约束大模型翻译对非标准表达有明显增益，特别是在多症状和否定表达上；但总体规范术语召回81%，尚未达到95%目标。下一轮应优先修复方言、隐喻和ASR表达漏抽取，并单独校准Hard Negative门控。修复时应保留100条不参与调参的最终留出集，避免针对同一批300条过拟合。
